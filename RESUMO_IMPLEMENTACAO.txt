🚀 COMO INICIAR:

1. Backend:
   cd packages/backend
   npm run dev
   (rodará em http://localhost:3000)

2. Frontend:
   cd packages/frontend
   npm run dev
   (rodará em http://localhost:5173)

3. Acessar:
   http://localhost:5173/correcao

═══════════════════════════════════════════════════════════════════════════════

📝 EXEMPLO DE USO:

1. Prepare dois arquivos CSV:

   gabarito.csv:
   email,nome,Q1,Q2,Q3,Q4,Q5
   professor@teste.com,Gabarito,A,B,C,D,A

   respostas.csv:
   email,nome,Q1,Q2,Q3,Q4,Q5
   aluno1@teste.com,Aluno Um,A,B,C,D,A
   aluno2@teste.com,Aluno Dois,A,B,C,E,B
   aluno3@teste.com,Aluno Três,B,B,C,D,A

2. Acesse http://localhost:5173/correcao

3. Selecione os arquivos CSV

4. Digite um ID para a prova

5. Selecione o modo de correção

6. Clique em "Processar Correção"

7. Veja os resultados

8. Clique em "Ver Detalhes" para informações por aluno

9. Clique em "Exportar CSV" para salvar relatório

🎓 MODO DE CORREÇÃO RIGOROSA:

• Uma única resposta correta por questão
• Não aceita variações
• Usa algoritmo de decomposto em potências de 2
• Ideal para: Testes com resposta exata
• Exemplo: Se o gabarito é "A", só aceita "A"

═══════════════════════════════════════════════════════════════════════════════

🎓 MODO DE CORREÇÃO MENOS RIGOROSA:

• Aceita variações na ordem das respostas
• Ideal para questões com múltiplas respostas corretas
• Mais flexível
• Exemplo: Se o gabarito é "A,B", aceita "B,A"

📊 ESTATÍSTICAS CALCULADAS:

O sistema calcula automaticamente:

Total de Alunos: Quantidade de registros processados
Média: Soma das notas / quantidade de alunos
Máximo: Maior nota obtida
Mínimo: Menor nota obtida
Mediana: Valor central (50º percentil)
Desvio Padrão: Dispersão das notas em relação à média

Fórmula da nota: (acertos / total_questões) * 10
